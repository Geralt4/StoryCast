# StoryCast App Store Deployment Plan

## Overview
Prepare StoryCast v1.1 for App Store submission by addressing compliance requirements, fixing the Tip Jar, and resolving configuration issues.

---

## Part 1: Critical Blockers (Required for App Store Approval)

### 1.1 Create Privacy Manifest (`PrivacyInfo.xcprivacy`)
**Priority**: Critical — App Store will reject without this

Apple requires a Privacy Manifest for apps using required-reason APIs. StoryCast uses `UserDefaults` (64+ occurrences across 14 files), which falls under `NSPrivacyAccessedAPICategoryUserDefaults`.

**Actions**:
- Create `StoryCast/PrivacyInfo.xcprivacy` with the following content:
  ```xml
  <?xml version="1.0" encoding="UTF-8"?>
  <!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
  <plist version="1.0">
  <dict>
      <key>NSPrivacyAccessedAPITypes</key>
      <array>
          <dict>
              <key>NSPrivacyAccessedAPIType</key>
              <string>NSPrivacyAccessedAPICategoryUserDefaults</string>
              <key>NSPrivacyAccessedAPITypeReasons</key>
              <array>
                  <string>CA92.1</string>
              </array>
          </dict>
      </array>
  </dict>
  </plist>
  ```
- Add the file to the Xcode project target

### 1.2 Add "Restore Purchases" to Tip Jar
**Priority**: Critical — Apple Review Guideline 3.1.1 requires this

The Tip Jar uses consumable IAPs. While consumables technically don't need restoration, Apple reviewers routinely reject apps without a visible restore mechanism.

**Actions**:
- Modify `TipJarView.swift` to add a "Restore Purchases" button
- Add `restorePurchases()` method to `TipJarManager.swift`:
  ```swift
  func restorePurchases() async {
      do {
          try await AppStore.sync()
          errorMessage = nil
      } catch {
          AppLogger.storeKit.error("Restore failed: \(error.localizedDescription)")
          errorMessage = "Could not restore purchases"
      }
  }
  ```
- Add restore button UI in `TipJarView.swift` (in a footer section or separate section)

### 1.3 Add Export Compliance Key to Info.plist
**Priority**: High — Avoids manual questionnaire on every submission

**Actions**:
- Add to `Info.plist`:
  ```xml
  <key>ITSAppUsesNonExemptEncryption</key>
  <false/>
  ```

## Part 2: Fix Tip Jar (Products Not Loading)

### 2.1 Create StoreKit Configuration File for Local Testing
**Priority**: High — Enables local testing without App Store Connect

**Actions**:
- Create `StoryCast/TipJar.storekit` with consumable products:
  ```plist
  <?xml version="1.0" encoding="UTF-8"?>
  <!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
  <plist version="1.0">
  <dict>
      <key>com.apple.in-app-purchases</key>
      <array>
          <dict>
              <key>ProductID</key>
              <string>tip_small</string>
              <key>ReferenceName</key>
              <string>Small Tip</string>
              <key>Type</key>
              <string>Consumable</string>
              <key>Price</key>
              <real>0.99</real>
          </dict>
          <dict>
              <key>ProductID</key>
              <string>tip_medium</string>
              <key>ReferenceName</key>
              <string>Medium Tip</string>
              <key>Type</key>
              <string>Consumable</string>
              <key>Price</key>
              <real>2.99</real>
          </dict>
          <dict>
              <key>ProductID</key>
              <string>tip_large</string>
              <key>ReferenceName</key>
              <string>Large Tip</string>
              <key>Type</key>
              <string>Consumable</string>
              <key>Price</key>
              <real>4.99</real>
          </dict>
      </array>
  </dict>
  </plist>
  ```
- Add to Xcode project
- Enable StoreKit configuration in the app's scheme (Edit Scheme → Run → Options → StoreKit Configuration)

### 2.2 Configure IAP in App Store Connect
**Priority**: High — Required for production

This must be done manually in App Store Connect. The code cannot automate this step.

**Steps** (user must perform):
1. Go to App Store Connect → My Apps → StoryCast
2. Navigate to Features → In-App Purchases
3. Create 3 consumable IAPs:
   - **tip_small**: Small Tip, $0.99
   - **tip_medium**: Medium Tip, $2.99
   - **tip_large**: Large Tip, $4.99
4. Add localized descriptions for each
5. Submit IAPs for review (can be done with app submission)

### 2.3 Add Better Error Handling / Loading States
**Priority**: Medium — Improves UX

**Actions**:
- Add a loading state while fetching products
- Show more descriptive error when products fail to load
- Add retry button when products fail to load
